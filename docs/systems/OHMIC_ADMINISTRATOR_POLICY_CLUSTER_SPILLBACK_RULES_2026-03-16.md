# Ohmic Administrator Policy Cluster Spillback Rules

Date: 2026-03-16
Project: ohmic

## Purpose

Define when administrator-heavy stacks should spill back excess policy-cluster
items.

## Spillback Triggers

- a branch is no longer adjacent to the primary policy lane
- the stack carries duplicate policy branches
- filing continuity is no longer improved by keeping the item local

## Rule

Spill back distant or redundant policy work before sacrificing the core filing
and routing continuity of the administrator stack.
