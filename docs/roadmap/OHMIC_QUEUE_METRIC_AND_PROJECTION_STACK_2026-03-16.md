# Ohmic Queue Metric And Projection Stack

## Purpose

Push the queue-capacity wave one rung closer to runtime use by defining the
dashboard-facing metrics, snapshot shape, and urgency signals that can drive
projection generation and operator visibility.

## Focus

### 1. Headroom Visibility

- headroom dashboard metrics
- queue-capacity dashboard cards
- same-family pressure rollups

### 2. Runtime Snapshot Shape

- queue health snapshot
- projection-friendly status bands
- refill urgency score

### 3. Follow-On Policy

- same-family floor tuning
- warning thresholds
- refill prioritization support

## Outcome Standard

If this stack lands, the queue-capacity model will have a usable runtime-facing
metric layer instead of stopping at policy-only docs.
