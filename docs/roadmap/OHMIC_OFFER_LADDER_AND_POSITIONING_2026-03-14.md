# Ohmic Offer Ladder And Positioning

## Objective

Turn the current `free / pro / hardware-linked` idea into a concrete outward
offer structure that matches the current product surfaces and gives the team a
clear conversion path to refine.

## Positioning Thesis

Ohmic should be positioned as:

- the practical audio design, measurement, and control platform
- free to learn and start
- paid when deeper workflows, persistence, support, and hardware-connected value
  become worth money

This is not just a content site and not just a hardware dream. It is a software
platform that earns trust through useful tools, then expands into premium
workflow and hardware-linked capability.

## Offer Ladder

### 1. Free

Working name:

- `Ohmic Free`

Job:

- attract new users
- teach them quickly
- let them solve a real first problem
- capture interest without making them pay before trust exists

Positioning:

Ohmic Free is the easiest way to start designing, learning, and validating an
audio system without committing to hardware or a subscription first.

Core value:

- open learning and search-entry content
- static calculators and tool pages
- suite hubs and educational paths
- basic account-level entry into store/help flows where appropriate

Current surfaces that fit:

- `/design/*`
- `/measurement/*`
- `/learn/*`
- `/design-suite`
- `/measurement-suite`
- `store.catalog`
- authenticated `community.request_help`

Primary CTA role:

- start with a tool
- learn with a guide
- create an account
- request help

### 2. Pro

Working name:

- `Ohmic Pro`

Job:

- monetize serious software users before hardware is required
- provide deeper workflow value, not just remove arbitrary locks

Positioning:

Ohmic Pro is for builders, installers, tuners, and serious hobbyists who want
faster workflows, better project continuity, stronger measurement/design depth,
and priority-level support over time.

Core value:

- advanced design and measurement workflows
- saved projects and stronger workflow continuity
- exports, premium workflow bundles, and richer gated features
- community share access and billing-management surfaces
- higher-touch support path than free

Current and near-current surfaces that fit:

- paid route and gate model already present in app/runtime
- `community.share`
- `billing.manage`
- premium workflow candidates from design/measurement/app surfaces

Primary CTA role:

- upgrade for better workflow
- save time
- save and manage projects
- unlock premium measurement or design capability

### 3. Hardware-Linked

Working name:

- `Ohmic Connected`

Job:

- connect the software platform to AmpLab-class hardware
- make hardware ownership feel like a higher-value operating mode, not a
  separate disconnected product

Positioning:

Ohmic Connected is the hardware-aware layer for users running Ohmic-native
devices who want control, commissioning, measurement, and device-linked premium
workflows in one system.

Core value:

- hardware control
- hardware-attested access
- device-aware workflows
- premium commissioning and measurement paths tied to Ohmic hardware
- future platform advantage as the hardware family grows

Current and near-current surfaces that fit:

- `hardware.control`
- AmpLab decks and device workflows
- measurement/control flows that benefit from hardware-bound trust
- future `dsp-wireless-controller`, `rta-spl-meter`, and related lanes

Primary CTA role:

- connect your hardware
- unlock device-aware workflows
- move from software utility into full system control

## Recommended First Paid Entry Point

Recommendation:

- make `Ohmic Pro` the first true commercial push

Why:

- it does not require hardware availability to work
- it aligns with the current app/runtime and static tool base
- it supports the thesis that software funds hardware
- it gives a clear upgrade path from free search/content/tool traffic

The first paid promise should be:

- deeper workflow
- better continuity
- better support

not:

- vague exclusivity
- paywalling the whole learning surface
- pretending hardware is ready before it is

## Recommended First CTA Surfaces

### Free To Pro

- suite hubs: `/design-suite`, `/measurement-suite`
- static tool pages with clear next-step prompts
- app entry/dashboard surfaces where advanced workflow value is visible
- community/help surfaces after a user shows serious intent

### Pro To Hardware-Linked

- hardware and platform pages
- app/device setup surfaces
- measurement and control workflows where device linkage adds real value

## Audience Fit

### Best initial fit for Free

- curious builders
- learning-stage hobbyists
- SEO-discovered users
- people with one immediate problem to solve

### Best initial fit for Pro

- repeat builders
- installers
- advanced hobbyists
- tuners and measurement-focused users

### Best initial fit for Hardware-Linked

- early adopters of AmpLab-class hardware
- users wanting integrated control and measurement
- field users who benefit from attested device-aware workflows

## Guardrails

Do not position `Pro` as:

- merely fewer restrictions
- admin/debug access
- a vague “support us” tier with no real workflow value

Do not position `Hardware-Linked` as:

- the only serious version of Ohmic
- a separate brand universe disconnected from the software ladder

Do position the ladder as:

- learn and solve first
- go deeper when the workflow matters
- connect hardware when the system value is real

## Recommended Copy Direction

### Free headline direction

- Learn, design, and measure before you spend blindly.

### Pro headline direction

- Upgrade from helpful tools to a serious workflow.

### Hardware-linked headline direction

- Bring design, measurement, and device control into one system.

## Review Questions

- should `hardware-linked` stay a descriptive tier name, or eventually become a
  more product-like named membership
- should the first `Pro` promise center more on design, more on measurement, or
  explicitly on both
- should paid support be bundled into `Pro` immediately or staged as a later
  enhancement

## Recommendation

Use this ladder as the baseline for:

- offer-page copy
- upgrade CTA design
- billing and gate alignment
- channel messaging

Then refine the tier promises only after the first public positioning pass is
written and critiqued.
