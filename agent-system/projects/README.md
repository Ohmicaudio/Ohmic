# Project Overlays

Each file in this folder gives the shared system a concise cross-session view of one repo or product lane.

## Purpose

Project overlays sit between global memory and repo-local docs.

They should answer:

- what this project is
- where the working repo lives
- what is currently true
- what major drift or risk exists
- which local docs matter first

## Good Overlay Shape

- short identity section
- current repo/root path
- current truth
- known drift
- skills
- keywords
- access
- first-read docs
- next move

Keep overlays factual and compact.

Use the shared section names from `instructions/voice-and-format.md` and start from `templates/project-overlay-template.md` when possible.
