# Terminology

## Status Labels

- `canonical`
  - current source of truth
- `working`
  - current editable state, not yet fully settled
- `reference`
  - useful to consult, not authoritative
- `deprecated`
  - kept for history, not for active decisions
- `archive`
  - retained history or source material not meant for active editing

## Memory Horizons

- `short-term`
  - active, rewriteable, near-horizon work
- `mid-term`
  - current initiatives and unresolved multi-session work
- `long-term`
  - durable facts that should survive many sessions

## Agent Terms

- `remote agent`
  - cross-session, cross-project continuity and synthesis role
- `local agent`
  - repo-grounded execution and verification role

## Shared Memory Fields

- `skills`
  - capabilities, experience areas, or competencies relevant to current or durable work
- `keywords`
  - short retrieval terms that help route future search and recall
- `access`
  - verified tools, credentials, services, or local capabilities available to agents
- `watchouts`
  - fresh mistakes, fragile areas, or current failure patterns
- `lessons`
  - repeated patterns that should influence future behavior
- `next move`
  - clearest next action from the current state

## Project Names

- `Ohmic`
  - umbrella/context repo and workspace root
- `ohmic-audio-labs`
  - main web/app/backend product repo
- `cyd-remote`
  - Cheap Yellow Display handheld remote repo
- `amplab-firmware`
  - preferred umbrella name for the firmware lane, subject to final repo naming if needed

## Naming Rule

When a folder or file label stops matching reality, update the label or add a pointer note quickly. Do not let misleading names become policy.
