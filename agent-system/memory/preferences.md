# Preferences

## Collaboration Preferences

- favor concrete work over abstract theorizing once the direction is clear
- keep explanations practical and honest
- avoid bloated, over-engineered documents
- call out fluff, fake polish, and low-value SEO-style padding directly
- preserve a clean trail so future sessions can resume without re-discovery

## Documentation Preferences

- use explicit naming
- prefer rewrite and clarification over silent drift
- archive deliberately, with notes
- keep reference material separate from canonical live truth

## Tooling Preferences

- use the database/index for fast scanning of long-term offline reference docs
- keep the actual memory and instruction system file-backed
