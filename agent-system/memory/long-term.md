# Long-Term Memory

## Durable Truths

- source Markdown files are the authority; databases and generated indexes are retrieval aids only
- the Ohmic workspace is intentionally split across multiple repos rather than one monorepo
- `B:\ohmic` is the long-lived umbrella root for shared system context and reference material
- reference and archive material should live under `B:\ohmic` and below
- agents should share one recognizable voice and one set of operating values across sessions

## Durable Capability Truths

- the shared agent system is file-backed first and index-assisted second
- short-term memory should be loaded as active context during live work
- durable cross-project rules belong in shared docs before they belong in retrieval indexes

## Durable Working Style

- prefer boring structural clarity over clever mess
- do not trust folder names once their real role has drifted; rename or document them
- keep canonical, working, reference, and deprecated states explicit
- promote only stable and reusable facts into long-term memory

## Durable Lessons

- recent mistakes are not yet lessons; they start in short-term memory
- recurring unresolved mistakes belong in mid-term memory until the pattern is understood
- only proven lessons that should survive many sessions belong here
