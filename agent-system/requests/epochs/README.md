# Queue Epochs

Use this folder to record project pivots that change queue reliability.

Create an epoch record when:

- the active product repo changes
- the execution plane changes enough to invalidate older queue wording
- a new runtime or contract layer becomes the real work surface
- older `ready/` packets may still be useful, but no longer safe to trust as
  current without re-review

Queue epochs are not requests and not claims.
They are pivot markers that tell the queue how to age safely.
