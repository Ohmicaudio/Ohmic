# Define Administrator Approval Requirement Matrix

## Goal

Define the matrix that determines which administrator actions are freely
allowed, approval-gated, or disallowed by overlay and intake context.

## Focus

- action-by-action approval requirements
- intake-kind sensitivity rules
- overlay-specific gating
- default-safe behavior
- rejected path behavior

## Acceptance

- one approval-matrix packet is explicit
- approval requirements are separated from generic action availability
