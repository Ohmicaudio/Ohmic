# Define Administrator Intake Status Lifecycle

## Goal

Define the stable state lifecycle for an administrator intake item from capture
through routing, hold, archive, and reprocess states.

## Focus

- intake lifecycle states
- allowed transitions
- hold and wait states
- routed versus archived separation
- normalization-warning branches

## Acceptance

- one intake lifecycle packet is explicit
- status names support desk use and automation equally
- invalid or ambiguous transitions are called out
