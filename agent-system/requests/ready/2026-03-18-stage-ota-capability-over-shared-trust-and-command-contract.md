# Stage OTA Capability Over Shared Trust And Command Contract

## Goal

Slot OTA behind BLE/setup/auth and network bootstrap so updates become a normal
trusted capability instead of a one-off updater path.

## Acceptance

- OTA has a defined place in the runway
- OTA depends on the shared trust/authority floor
- status, apply, failure, and reboot truth are explicit capability concerns
