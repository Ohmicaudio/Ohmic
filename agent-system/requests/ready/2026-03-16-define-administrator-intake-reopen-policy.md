# Define Administrator Intake Reopen Policy

## Goal

Define the explicit policy for reopening archived or routed intake items back
into active administrator flow.

## Focus

- reopen triggers
- required operator intent
- audit requirements
- status restoration target
- restriction cases

## Acceptance

- one reopen-policy packet is explicit
- archived and routed items do not silently drift back into the queue
