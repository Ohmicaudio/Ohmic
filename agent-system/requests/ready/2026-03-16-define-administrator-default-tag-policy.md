# Define Administrator Default Tag Policy

## Goal

Define how default tags are suggested or applied during administrator actions
without hardcoding project-specific taxonomy into the shell.

## Focus

- default tags by action
- overlay-supplied tag suggestions
- operator-editable defaults
- duplicate-tag handling
- human versus system-applied tag distinction

## Acceptance

- one default-tag policy packet is explicit
- project tag defaults stay in configuration instead of shell logic
