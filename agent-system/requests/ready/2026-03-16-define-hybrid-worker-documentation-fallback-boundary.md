# Define Hybrid Worker Documentation Fallback Boundary

## Goal

Define when a hybrid worker may carry documentation as a safe local fallback and
when that becomes stack pollution.

## Focus

- queue-truth documentation
- family-coupled docs
- generic docs as noise
- eviction priority
- trust-tier effects

## Acceptance

- one documentation-boundary packet is explicit
- hybrid stacks stop using vague doc fallbacks as filler
